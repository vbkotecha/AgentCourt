# AgentCourt Token Launch Strategy — June 20, 2026

## WHY A TOKEN?
- AgentCourt needs economic incentives for: dispute submission, ruling quality, precedent creation
- Token creates skin-in-the-game for agents submitting disputes and judges providing rulings
- Community governance over dispute templates and protocol upgrades
- Revenue sharing: x402 micropayments flow to token holders

## PLATFORM OPTIONS

### Option 1: Clanker (Base)
- **Pros:** Instant deployment, auto liquidity pool, SDK available, multi-chain (Base, Arbitrum, Monad)
- **Cons:** Farcaster-centric community, meme-token reputation, limited customization
- **Launch method:** Tag @clanker on Farcaster, use deploy wizard, or SDK
- **Contract:** 0xE85A59c628F7d27878ACeB4bf3b35733630083a9 on Base
- **Time to launch:** < 1 minute
- **Cost:** Gas only

### Option 2: Virtuals Protocol
- **Pros:** Purpose-built for AI agents, 44K+ agent projects, tokenized AI labor economy
- **Cons:** More complex, longer setup, competitive landscape
- **Launch method:** Create agent on Virtuals, bond token through their protocol
- **Time to launch:** Hours to days
- **Cost:** Bonding curve minimum

### Option 3: Custom ERC-20 on Base
- **Pros:** Full control, custom tokenomics, no platform dependency
- **Cons:** No built-in community, need to bootstrap liquidity manually
- **Launch method:** Deploy contract, add liquidity on Uniswap
- **Time to launch:** Days
- **Cost:** Development + liquidity
